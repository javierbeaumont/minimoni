# ADR-0003: tomlc17 as the configuration parser

**Date:** 2026-05-18
**Status:** Accepted

## Context

minimoni uses TOML for configuration. The config includes a `[[alert]]` array-of-tables
block (users can define multiple alerts), which is non-trivial to parse correctly by hand.
The parser must vendor as two files, be GPLv3+ compatible, and add minimal binary size.

## Alternatives considered

| Option | Reason rejected |
|---|---|
| inih | INI has no standard array syntax — multiple alerts would need `[alert.0]`, `[alert.1]`… |
| Hand-rolled parser | `[[alert]]` array-of-tables is non-trivial; significant bug surface |
| tomlc99 | Archived (unmaintained) — tomlc17 is the maintained successor by the same author |

## Decision

Use **tomlc17 R260517** (MIT), the maintained successor to tomlc99 by CK Tan. Full TOML v1.1
support in exactly two files (`tomlc17.c` + `tomlc17.h`). ~32 KB compiled contribution.

## Consequences

- `[[alert]]` array-of-tables handled correctly with no format hacks.
- ~32 KB binary contribution (negligible).
- Two-file vendor: updating means replacing `tomlc17.c` and `tomlc17.h`.
